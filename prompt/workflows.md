# 对话流程图解

> 辅助理解智能体的全部分支逻辑，便于调试和迭代。

---

## 一、入口意图分类

```
用户输入
    │
    ├─ "卖" / "发布" / "转卖" / "出售" / "出" → 卖家流程
    │
    ├─ "找" / "买" / "搜" / "有没有" / "需要" → 买家流程
    │
    ├─ 点击/输入标签名（"文具""书""生活用品"等）→ 标签浏览
    │
    ├─ "下架" / "卖掉了" / "删" / "已出" → 状态更新
    │
    ├─ "规则" / "安全" / "怎么交易" → 安全说明
    │
    ├─ "帮助" / "功能" / "你会什么" → 功能介绍
    │
    └─ 其他/模糊 → 追问意图
```

## 三、买家搜索流程图

买家有两条路径入口：点击/输入标签 → 标签浏览；自由描述 → 语义搜索。

```
用户触发买家流程
    │
    ├── 点击标签 / 输入标签名 ──────────────────────────┐
    │   （如"文具""书"）                                 │
    │   ▼                                               │
    │  search_by_tag("文具")   ← 精确搜索               │
    │   │                                               │
    │   ├── 有结果 → 展示标签卡片列表                    │
    │   │          → "想了解哪个？详情1/2/3"             │
    │   │          → "看看其他热门标签？"                │
    │   │                                               │
    │   └── 无结果 → "还没有这个标签的物品～"            │
    │              → 推荐其他热门标签                    │
    │                                                   │
    └── 自由描述需求 ──────────────────────────────────┐
        （"我需要一辆通勤用的自行车"）                    │
        ▼                                              │
    ┌──────────────────────────────────┐                │
    │ 判断描述清晰度                    │                │
    │  ┌──────────┐  ┌──────────────┐  │                │
    │  │ 清晰     │  │ 模糊         │  │                │
    │  └────┬─────┘  └──────┬───────┘  │                │
    │       ▼                ▼         │                │
    │  提取关键词          追问2-3个   │                │
    │  调用 search_item()  问题细化    │                │
    │                       │         │                │
    │                       ▼         │                │
    │                  提取关键词      │                │
    │                  调用search_item│                │
    └──────────────────────────────────┘                │
        │                                              │
        ▼                                              │
    ┌──────────────────────────────────┐                │
    │ 展示结果                          │                │
    │  ┌────────────────────┐          │                │
    │  │ 有匹配结果          │          │                │
    │  │ Top3-5卡片展示     │          │                │
    │  └────────────────────┘          │                │
    │  ┌────────────────────┐          │                │
    │  │ 无匹配结果          │          │                │
    │  │ "换个关键词/试试    │          │                │
    │  │  热门标签？"       │          │                │
    │  └────────────────────┘          │                │
    └──────────────────────────────────┘                │
        │                                              │
        └── 两条路径在此汇合 ───────────────────────────┘
                        │
                        ▼（用户选了一个）
                ┌──────────────────────────┐
                │ get_item_detail(item_id) │
                │ 展示完整信息 + 联系方式  │
                │ 附交易安全提示           │
                └──────────────────────────┘
```

## 四、工具调用时机一览

| 工具 | 触发条件 | 调用参数 |
|------|---------|---------|
| `list_item` | 卖家确认发布 | 物品名、描述、tags、联系方式等 |
| `search_item` | 买家描述需求 | 查询文本 |
| `search_by_tag` | 用户点击/输入标签名 | 标签名称（如"文具"） |
| `get_popular_tags` | 开场时/无匹配时推荐 | 无需参数 |
| `analyze_image` | 卖家上传照片 | 图片 URL |
| `web_search` | 卖家选择"AI生成介绍" | 物品名称 |
| `get_item_detail` | 买家查看详情 | 物品 ID |
| `update_status` | 卖家标记已售/下架 | 物品 ID + 新状态 |

## 五、错误恢复路径

```
┌─ 场景A：用户在中途改变意图 ──────────────────────┐
│ "等等，我不卖了，我想买东西"                        │
│ 处理：保存当前进度 → 切换到目标流程                  │
└──────────────────────────────────────────────────┘

┌─ 场景B：用户输入不完整信息 ───────────────────────┐
│ 用户只说了"卖自行车"就停了                         │
│ 处理：继续追问缺失信息（照片、描述、联系方式）        │
│ "好的自行车！请问有照片吗？或者我先记下信息？"       │
└──────────────────────────────────────────────────┘

┌─ 场景C：用户提供联系方式后撤回 ───────────────────┐
│ "算了，我暂时不想卖了"                             │
│ 处理：不保存任何信息 → "好的，随时想卖找我～"       │
└──────────────────────────────────────────────────┘

┌─ 场景D：用户想多次发布 ──────────────────────────┐
│ "我还有一个台灯要卖"                              │
│ 处理：重新进入卖家流程                              │
└──────────────────────────────────────────────────┘
```
