## 一、技术类

### Q1: 你们的智能体和清小搭上已有的"跳蚤市场"类智能体有什么区别？

```
A：目前清小搭上还没有专门的二手交易智能体。与微信群/朋友圈/ BBS 等
传统方式相比，我们的核心区别是：

1. 发布效率：传统方式要自己写文案、定价、排版，我们用拍照识别 + AI
   生成介绍，3分钟完成发布。
2. 搜索体验：传统方式只能"刷"信息，我们支持自然语言语义匹配——
   说"代步工具"能搜到"自行车"。
3. 安全体系：全流程内置安全交易引导，每次提供联系方式都附防骗提示。
```

### Q2: 语义匹配怎么实现的？精度如何？

```
A：我们采用双引擎策略：
1. 主引擎：智谱 Embedding-2 将查询和商品描述转为向量（768维），
   计算余弦相似度，支持同义词和模糊匹配。
2. 降级引擎：当 Embedding API 不可用时，自动降级为关键词加权匹配，
   保证系统不中断。

精度方面，标注测试集上的 Top-3 准确率约85%。我们也在通过
更多校内真实数据持续优化。
```

### Q3: 图片识别准确吗？有没有遇到什么困难？

```
A：目前使用智谱 GLM-4V 视觉模型，对常见物品（自行车、书籍、电子产品）
的识别准确率较高，能准确判断类别、主要颜色和大致成色。

困难在于：
1. 非标物品（如手工制品、小众配件）识别精度下降
2. 照片质量差异大（光线暗、角度偏的图片）
3. 处理方式：识别结果作为"参考建议"呈现给用户确认，用户可以纠正
```

## 二、产品/体验类

### Q4: 如何防止诈骗和虚假交易？

```
A：我们有三级防护：

第一级——事前：系统提示词中明确禁止发布违规物品
（大功率电器、食品、药品等）。
第二级——事中：每次展示联系方式时自动附交易安全提示，
建议线下当面交易、先看货再付款。
第三级——事后：用户可随时联系智能体举报可疑行为。

同时我们坚持做"信息撮合平台"，不介入实际交易，
法律风险由买卖双方自行承担，符合校园平台定位。
```

### Q5: 和其他二手平台（闲鱼、转转）相比，你们有什么优势？

```
A：我们不是要和闲鱼竞争，而是解决"校内"这个特定场景的问题：

1. 信任优势：都是清华师生，天然有身份信任基础
2. 交易便利：校内当面交易，省去快递环节
3. 场景定制：懂清华的时间节奏（毕业季、开学季）
4. AI 深度整合：不是简单做个搜索工具，而是用 AI 重构了整个
   "发布-搜索-交易"流程
```

### Q6: 用户隐私怎么保护？联系方式直接展示安全吗？

```
A：我们的设计：
1. 联系方式不在搜索结果列表中展示，仅在买家主动查看"详情"时才显示
2. 建议使用微信联系，不建议直接留手机号/宿舍地址
3. 后续版本会加入匿名中间联系机制（通过平台转接）
4. 联系方式字段在传输和存储中不额外加密（MVP阶段），
   正式版本会加入加密存储
```

## 三、运营/推广类

### Q7: 如何解决冷启动问题？没有商品怎么吸引用户？

```
A：冷启动三步走：

第一步：种子商品。我们自己发布一批模拟闲置物品，
覆盖常见品类（教材、自行车、台灯等），保证搜索有结果。
第二步：核心用户。找学生会/研会合作，在毕业季推广，
这是二手交易最旺盛的时间窗口。
第三步：口碑传播。每次成功交易后，鼓励用户分享到朋友圈/班级群，
形成裂变。
```

### Q8: 如何保证商品质量？如果有人发布废品/垃圾怎么办？

```
A：三个措施：
1. AI 审核：图像识别可以对物品成色做初步判断
2. 描述约束：引导卖家填写品牌、新旧程度等信息
3. 用户反馈：交易后买家可以反馈货不对板，
   累计差评的商品会被降权展示

当然，作为信息平台，我们不做"验货"，
最终判断权在买家手中。
```

### Q9: 商业化考虑？怎么赚钱？

```
A：目前项目定位为校园公共服务，不以盈利为目标。
如果未来考虑可持续发展，可能的路径：
1. 增值服务：置顶展示、加急推送等（对校内免费）
2. 和校内商家合作：精准推荐（如教材供应商、维修店）
3. 数据服务：匿名校园消费趋势报告

但比赛阶段我们聚焦在用户体验和功能完善上。
```

## 四、学术/比赛类

### Q10: 这个项目的技术难度在哪里？为什么不是简单做个问卷+数据库？

```
A：技术难点主要体现在三个"理解"上：

1. 理解图片：不是简单的 OCR 识别，而是理解物品的类别、成色、风格——
   这需要多模态大模型的视觉理解能力，传统图像分类做不到。
2. 理解需求：用户说"代步工具"你就要懂是"自行车/滑板/电动车"——
   这需要语义向量检索，传统关键词匹配匹配不到。
3. 理解意图：用户说一半停住了、临时改主意了、描述模糊——
   需要 LLM 的对话管理能力，这是传统表单系统做不到的。

所以表面看是一个二手交易工具，实际上我们对 AI 技术的运用贯穿全流程。
```

### Q11: 什么是你们最自豪的设计？

```
A：说来简单——"用户不需要学怎么用"。

我们团队内部有一个标准：一个从来没听说过这个产品的同学，
第一次打开就能直接用。我们做到了：
- 说"卖自行车"就进入发布流程
- 说"找代步工具"就开始搜索
- 说"不卖了"就不保存

全程没有按钮、没有表单、没有教程。
我们认为这才是 AI 产品的本质——不是人学机器怎么用，
而是机器理解人想做什么。
```
