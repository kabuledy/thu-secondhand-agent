## 二、项目背景

### 问题发现

清华大学校园内二手物品交易需求旺盛，但存在以下痛点：

1. **信息分散**：闲置信息散落在朋友圈、微信群、BBS，查找困难
2. **匹配低效**：找不到合适的搜索方式，只能靠"刷"信息
3. **发布门槛**：写描述、拍照片、定价——流程繁琐，很多人懒得发
4. **安全隐忧**：缺少交易引导和风险提示

### 目标

打造一个"会思考的二手管家"——让发布像聊天一样简单，让搜索像说话一样自然。

## 三、核心功能

### 3.1 卖家端：AI 辅助发布

```
输入物品名 → 可选上传照片（AI自动识别） → 生成描述（可选） → 留联系方式 → 一键发布
```

创新点：
- **拍照识物**：上传照片后视觉模型自动分析物品类别、颜色、成色，减少填写工作量
- **AI 生成介绍**：联网搜索物品信息后，自动生成一段介绍草稿，用户确认或修改即可
- **引导式交互**：不需要学习操作，像聊天一样完成发布

### 3.2 买家端：语义搜索

```
自然语言描述需求 → 智能解析意图 → 按符合度排序推荐 → 查看详情
```

创新点：
- **模糊语义匹配**：搜索"代步工具"能匹配到"自行车"，不需要精确关键词
- **多维度排序**：综合语义相似度 + 关键词匹配 + 价格匹配 + 新鲜度
- **智能追问**：描述模糊时主动追问，帮你明确需求

### 3.3 安全交易引导

- 每次展示联系方式时附交易安全提示
- 建议线下当面交易、选择校内公共场所
- 禁止违禁物品发布

## 四、技术方案

### 4.1 整体架构

```
清小搭智能体（LLM）
    │ （通过工具函数调用）
后端 API（Python Flask）
    │
    ├── 语义引擎（Embedding + 余弦相似度）
    ├── 视觉引擎（GLM-4V 图像分析）
    └── 搜索引擎（联网搜索辅助生成）
```

### 4.2 关键技术

| 技术 | 用途 | 方案 |
|------|------|------|
| LLM | 对话理解与生成 | 清小搭平台内置 |
| 语义匹配 | 模糊搜索 | 智谱 Embedding-2 |
| 图像识别 | 拍照识物 | 智谱 GLM-4V |
| 数据存储 | 商品信息持久化 | Supabase / JSON |
| 部署 | API 服务 | 腾讯云函数 |

### 4.3 数据结构

```
item {
    item_id:     唯一编号
    name:        物品名称
    description: 详细描述
    price:       价格
    category:    自动分类
    tags:        关键词标签
    image_url:   图片链接
    image_description: 图片分析结果
    contact:     联系方式
    status:      状态（在售/已售/下架）
    created_at:  发布时间
}
```

## 五、项目亮点

1. **零学习成本**：聊天式交互，不需要学任何操作
2. **AI 赋能全流程**：拍照识物、自动生成介绍、语义匹配——每一个环节都有 AI 辅助
3. **贴近校园场景**：针对清华师生设计，交易安全提示切合实际
4. **可扩展性强**：后期可加入技能交换、拼单团购、课程资料共享等

## 六、社会价值

- 促进校园资源循环利用，减少浪费
- 降低学生交易成本，提高闲置物品流通效率
- 积累校内交易数据，为校园治理提供参考
