## 第一部分：这个项目到底是什么？

### 一句话版本

一个 AI 聊天机器人，清华同学可以在里面买卖二手物品。

### 具体一点

你用过 ChatGPT 对吧？"清小闲" 就是一个**垂直领域的 AI 助手**——它的功能被限定在校园二手交易上：

- 你说 **"我想卖自行车"** → 它会引导你填价格、联系方式 → 帮你发布
- 你说 **"我想找自行车"** → 它会去数据库里搜 → 把匹配的结果展示给你
- 你说 **"下架"** → 它要求你提供物品编号 → 验证通过后帮你下架

但它不是直接和你对话的。整个流程是：

```
你（在微信/网页上打字）
    ↓
清小搭平台（清华自己的 AI 平台，类似微信里的 AI 助手）
    ↓ 把你的话传给我写的后端程序
我写的后端程序（运行在阿里云服务器上）
    ↓ 调用 DeepSeek 大模型（类似 ChatGPT）
    ↓ 大模型决定要调用哪个功能
    ↓ 程序执行功能（查数据库/发布商品）
    ↓ 把结果返回给清小搭
清小搭平台把答案展示给你
```

### 为什么需要这么复杂？不能直接在 ChatGPT 里做吗？

因为 ChatGPT 没有"记忆"（不知道你之前说过什么），也没有"数据库"（不知道有哪些商品在卖）。所以我们需要：

1. **后端程序** — 处理逻辑、读写数据库
2. **数据库** — 存储商品信息
3. **大模型** — 理解自然语言、决定调用什么功能
4. **云服务器** — 让程序 24 小时运行，大家都能访问

## 第三部分：每个文件"为什么"要这么写

### 📄 prompt/system_prompt.md — AI 的灵魂

**这是什么：** 一份给 AI 的"岗位说明书"。

**为什么需要它：** 大模型（DeepSeek）本身什么都会，但我们需要把它限定在"校园二手助手"这个角色里。就像你招了一个实习生，你得告诉他：

- "你的身份是校园二手助手"
- "你的工作流程是：先问用户要卖什么，再问价格……"
- "你有这些工具可以用：发布商品、搜索商品……"

**怎么写的：**

```
## 你是谁
你是清小闲，清华大学校园二手智能助手。

## 可用工具
| 工具名 | 用途 | 什么时候用 |
|--------|------|-----------|
| list_item | 发布商品 | 卖家确认发布时 |
| search_item | 搜索商品 | 买家描述需求时 |
| ...

## 对话流程
用户说"我想卖自行车" → 走卖家流程 → 问价格 → 问联系方式 → ...
```

这份文件会被 `agent.py` 在程序启动时读取，然后塞给 DeepSeek 大模型，让它按照这个规则来工作。

---

### 📄 backend/api/agent.py — 核心引擎

**这是什么：** 整个项目最重要的文件。它负责：

1. 读取 `system_prompt.md`
2. 把用户的话 + system prompt 一起发给 DeepSeek 大模型
3. 解析 DeepSeek 的回复——如果 DeepSeek 说"调用发布商品工具"，就去执行
4. 把执行结果返回给用户

**核心概念：Function Calling（函数调用）**

这是大模型的一个关键能力。普通的 ChatGPT 只能"说话"。但通过 Function Calling，大模型可以"做事"：

```
用户说："我要卖一辆自行车，300块"

DeepSeek 大模型分析后，内心想：
  → "用户要发布商品"
  → "我该调用 list_item 这个工具"
  → 返回一个特殊的指令：{ "tool": "list_item", "参数": {...} }

agent.py 收到这个指令后：
  → 调用 list_item.py 里的代码
  → 把商品写入数据库
  → 把结果返回给 DeepSeek
  → DeepSeek 生成最终回复："发布成功！编号是xxx"
```

**为什么需要 10 轮循环（MAX_TOOL_CALLS = 10）：**

因为有时候 AI 需要连续做好几件事。比如：

```
用户："我想卖自行车"
  → 第1轮：AI 问 "多少钱？"
  → 第2轮：用户说 "300"
  → 第3轮：AI 问 "联系方式？"
  → 第4轮：用户说 "微信 xxx"
  → 第5轮：AI 问 "标签？"
  ...
  → 第8轮：AI 终于调用 list_item() 发布
```

每一轮都是一次"发消息给 DeepSeek → 等回复"的过程。最多 10 轮防止死循环。

### 📄 backend/main.py — 程序的"大门"

**这是什么：** 一个用 Flask 框架写的 Web 服务器。

**类比：** 就像一家餐厅的前台。所有客人（网络请求）先进来，前台根据客人的需求：

- "我要结账" → 带到收银台（`/api/list_item`）
- "我要点菜" → 带到点餐区（`/api/search_item`）
- "我要投诉" → 带到经理室（`/v1/chat/completions`）

**代码里做了什么：**

```python

    # 接收用户的消息
    # 交给 agent.py 处理
    # 返回 AI 的回复
```

```python
@app.route("/api/list_item", methods=["POST"])
def list_item():
    # 这是直接发布商品的接口
    # 收到商品信息 → 交给 list_item.py → 写入数据库
```

**为什么有些接口有 Bearer Token 认证：**

### 📄 backend/api/database.py — 数据仓库管理员

**这是什么：** 操作 SQLite 数据库的代码。

**数据库是什么：** 简单理解为 Excel 表格，但程序可以自动读写。

```
items 表（商品表）：
┌─────────────┬──────────┬───────┬──────────┐
│ item_id     │ name     │ price │ status   │
├─────────────┼──────────┼───────┼──────────┤
│ ITEM-xxx001 │ 自行车   │ 300   │ active   │
│ ITEM-xxx002 │ 笔记本   │ 15    │ sold     │
└─────────────┴──────────┴───────┴──────────┘
```

**database.py 负责：**
- 创建表（`init_db()`）
- 添加商品（`add_item()`）
- 查询商品（`get_item()`）
- 更新状态（`update_item_status()`）
- 清空数据（`clear_all_items()`）

### 📄 backend/api/list_item.py — 发布商品

**逻辑流程：**
1. 收齐所有必要信息（名称、价格、联系方式、标签）
2. 校验格式（价格不能为空、标签最多5个）
3. 生成唯一编号（`ITEM-YYYYMMDD-XXXX`）
4. 写入数据库
5. 返回成功消息 + 物品编号

---

### 📄 backend/api/search_item.py — 搜索商品

**逻辑流程：**
1. 接收用户输入：`"我需要一辆通勤用的自行车，预算300"`
2. 解析查询：提取关键词`["自行车","通勤"]`、价格上限`300`
3. 从数据库取出所有在售商品
4. 对每个商品打分：
   - 名称匹配关键词 → +0.4分
   - 描述匹配关键词 → +0.2分
   - 价格在预算内 → +0.15分
   - 发布时间近 → +0.15分
5. 按总分排序，返回 Top 10

---

### 📄 backend/api/conversation_store.py — 对话记忆

**为什么需要它：** 清小搭平台每次只发"当前这一句话"，不会带之前的聊天记录。所以需要在后端自己记住。

**怎么做到的：** 每个对话有一个唯一 ID。每次用户说话，后台就把这句话存起来。下次用户再说话时，先把历史记录找出来，一起发给大模型。

---

### 📄 sync.py / sync_data.py / reset_data.py — 运维工具

| 文件 | 作用 | 什么时候用 |
|------|------|-----------|
| `sync.py` | 把本地代码上传到云服务器 | 修改了代码之后 |
| `sync_data.py` | 把本地数据库同步到云服务器 | 本地测试了大量数据后 |
| `reset_data.py` | 清空本地+云服务器的所有数据 | 想从头开始 |

**sync.py 的原理（重要知识点）：**

它用了一个叫 `paramiko` 的 Python 库，通过 SSH 协议连接到云服务器。

SSH 就像"远程桌面"的命令行版本——你可以在一台电脑上远程操作另一台电脑。

```
你本地的电脑 → SSH 连接 → 阿里云服务器
  (修改了代码)               (执行命令、替换文件)
```

## 第四部分：为什么需要云服务器？

### 问题

你的程序在你的电脑上跑着。但如果：

- 你的电脑关机了呢？
- 你的同学也想用这个程序呢？
- 清小搭平台需要 24 小时都能调用 API 呢？

### 解决方案：云服务器

云服务器就是"一直开着机、一直连着网"的远程电脑。

```
阿里云 ECS（Elastic Compute Service）
  → 一台放在阿里机房的电脑
  → 24小时开机
  → 有固定的公网 IP 地址（39.106.1.145）
  → 你通过 SSH 远程操作它
```

**类比：**
- 你的笔记本电脑 = 家里的厨房（只有你能用）
- 云服务器 = 餐厅的后厨（全天营业，大家都能来）

### 我们的服务器上跑了什么？

```
阿里云服务器 (39.106.1.145)
  │
  ├── Nginx（Web 服务器软件）← 监听 80 端口
  │   ├── 收到 "http://39.106.1.145/" 的请求
  │   ├── 如果是 /v1/ 开头的 → 转发给 Flask
  │   └── 如果是 /api/ 开头的 → 转发给 Flask
  │
  └── Flask（我们的 Python 程序）← 监听 5000 端口
      ├── 处理聊天请求
      ├── 读写数据库
      └── 调用 DeepSeek API
```

**端口是什么？** 想象一栋大楼（服务器），有不同的门（端口）：
- 80 号门（HTTP）：普通人访问网页走的门
- 443 号门（HTTPS）：加密的网页访问
- 5000 号门：我们的 Flask 程序在用的门，不对外开放（只让 Nginx 转发进来）

**Nginx 是什么？** 一个"门卫"软件：
1. 接收所有外来请求
2. 判断请求类型
3. 转发给正确的内部程序

## 第六部分：关键概念解释（大白话版）

### 什么是 API？

API = 应用程序编程接口。

**类比：餐厅的服务员**

- 你（用户）想点菜 → 告诉服务员（API）
- 服务员（API）→ 告诉后厨（程序）
- 后厨（程序）→ 做好菜 → 服务员端给你

在我们的项目里：
- `POST /api/list_item` ← 这是"点菜"的 API
- `POST /api/search_item` ← 这是"查菜单"的 API
- `POST /v1/chat/completions` ← 这是"和 AI 聊天"的 API

### 什么是 HTTP 方法？

- **GET** → 读取数据（像"看菜单"）
- **POST** → 创建数据（像"下单"）

### 什么是 JSON？

一种数据格式，人和程序都能看懂。

```json
{
  "name": "自行车",
  "price": "300",
  "tags": ["通勤", "交通"]
}
```

就像填写表格——每个字段（name）对应一个值（自行车）。

### 什么是 Function Calling（函数调用）？

大模型（DeepSeek、GPT）不仅能"说话"，还能"做事"。

普通模式：
```
你：今天天气怎么样？
AI：今天25度，晴天。
```

Function Calling 模式：
```
你：帮我查一下北京的天气
AI 内心想：→ 用户要查天气 → 应该调用 get_weather("北京")
            → 但我不会上网，需要代码帮我去查
AI 返回：{ 调用工具: "get_weather", 参数: { city: "北京" } }
代码执行：去天气网站查 → 返回结果给 AI
AI 最终说：北京今天25度，晴天。
```

在我们的项目里：
```
你：我想卖自行车，300块
AI 内心想：→ list_item 工具的参数是 name="自行车", price="300"
            → 但我自己不能写入数据库，要代码帮我写
AI 返回：{ 调用工具: "list_item", 参数: { name: "自行车", ... } }
代码执行：调用 list_item.py → 写入数据库 → 返回成功
AI 最终说：发布成功！编号是xxx
```

### 什么是 SQLite？

一个轻量级数据库，就是一个文件（`items.db`），不需要安装额外软件。适合小项目。

### 什么是 SSH？

Secure Shell。远程操作另一台电脑的"黑窗口"（命令行）。

```
ssh root@39.106.1.145
```

这行命令的意思是：通过 SSH 连接到 IP 为 39.106.1.145 的服务器，用 root 账号登录。

登录后你就可以：
- `ls` — 看文件列表
- `cat file` — 看文件内容
- `systemctl restart xxx` — 重启某个服务
- `rm file` — 删除文件

### 什么是 systemd？

Linux 系统的"任务管理器"。我们用 systemd 来管理 Flask 程序：

```bash
systemctl restart thu-secondhand.service  # 重启后端程序
systemctl status thu-secondhand.service   # 查看程序运行状态
```

这样即使程序崩溃了，systemd 会自动重启它。
