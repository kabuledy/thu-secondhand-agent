# 🖥️ 服务器终端操作教程——从零开始

> 本文面向**完全没用过终端的人**。
> 每一个命令都会拆开解释：为什么这么写、每个部分是什么意思、按哪个键。

---

## 零、最最基础的概念

### 什么是终端（Terminal）？

终端 = 一个**纯文字的操作界面**。没有按钮、没有鼠标、没有窗口，全靠**打字**来操作电脑。

就像电影里的黑客，在一个黑框框里敲命令——就是这个东西。

### 终端和 Windows 桌面的区别

| Windows 桌面（图形界面） | 终端（命令行） |
|------------------------|--------------|
| 双击文件夹 → 打开 | `cd 文件夹名` |
| 右键 → 删除 | `rm 文件名` |
| 鼠标拖拽复制 | `cp 来源 目标` |
| 看图软打开图片 | 终端不能直接看图（只看文字） |

### 打开本机 PowerShell

在 Windows 上：
1. 按键盘 `Win` 键（左下角）
2. 输入 `powershell`
3. 回车

会弹出一个**黑底蓝字**的窗口，这就是终端。

### 你会在两个终端之间切换

| 终端 | 在哪 | 操作的是哪台电脑 |
|------|-----|----------------|
| **本机 PowerShell** | 你自己电脑上打开的 | 你自己的电脑 |
| **SSH 远程终端** | 用 ssh 命令连上服务器后 | 阿里云服务器 |

**看提示符就能区分：**

```
# 本机 PowerShell（你在自己电脑上）
C:\Users\dengyi>

# SSH 远程终端（你在服务器上）
root@iZ2zebl4x12ziohbo8yqefZ:~#
```

## 一、如何连上服务器

### 第一步：打开本机 PowerShell

按 `Win` → 输入 `powershell` → 回车。

### 第二步：输入 SSH 命令

在 PowerShell 里**一个字不差**地输入：

```
ssh root@39.106.1.145
```

然后按 `回车`（Enter 键）。

### 这个命令拆开解释

```
ssh   root   @   39.106.1.145
 ↑     ↑     ↑        ↑
命令  用户名  分隔符  服务器的IP地址
```

意思是：**用 ssh 这个程序，以 root 身份，连接 IP 为 39.106.1.145 的电脑。**

### 第三步：输入密码

按完回车后，屏幕上会出现：

```
root@39.106.1.145's password:
```

这时候输入密码 `Thudy87ubantu...`

**⚠️ 重要：输入密码时屏幕不会显示任何东西（没有星号、没有光标移动），这是正常的。** 你只管继续打字，输完按回车。

### 第四步：连接成功

如果密码正确，你会看到：

```
Welcome to Alibaba Cloud Elastic Compute Service !
root@iZ2zebl4x12ziohbo8yqefZ:~#
```

这表示你已经连上了服务器。现在你输入的**所有命令都是在服务器上执行的**。

## 二、基础命令——在服务器上查看文件

### 2.1 查看当前位置（pwd）

```
pwd
```

按回车。

**pwd** = Print Working Directory（打印当前工作目录）

**作用：** 告诉你现在在哪个文件夹里。

```
root@iZ2zebl4x12ziohbo8yqefZ:~# pwd
/root
```

`/root` 就是你当前所在的位置（root 用户的家目录）。

---

### 2.2 查看当前文件夹有什么（ls）

```
ls
```

**ls** = List（列出）

**作用：** 列出当前文件夹里的所有文件和子文件夹。

```
root@iZ2zebl4x12ziohbo8yqefZ:~# ls
snap  thu-secondhand
```

看到两个东西：`snap` 文件夹（系统自带的）和 `thu-secondhand` 文件夹（我们的项目）。

#### 加参数查看详细信息

```
ls -lh
```

拆开：
- `ls` = 列出命令
- `-l` = long（长格式，显示更多信息）
- `-h` = human-readable（人类能看懂的大小，比如 4K、1.2M）

```
root@iZ2zebl4x12ziohbo8yqefZ:~# ls -lh
total 20K
drwxr-xr-x 2 root root 4.0K Jul  4 20:08 snap
drwxr-xr-x 6 root root 4.0K Jul  5 10:30 thu-secondhand
```

从左到右：文件权限、所有者、大小、修改时间、文件名。

---

### 2.3 进入文件夹（cd）

```
cd 文件夹名
```

**cd** = Change Directory（切换目录）

```
# 进入项目目录
root@iZ2zebl4x12ziohbo8yqefZ:~# cd thu-secondhand

# 看看里面有什么
root@iZ2zebl4x12ziohbo8yqefZ:~/thu-secondhand# ls
backend  chat.html  docs  prompt
```

#### 回到上一级

```
cd ..
```

`.` 代表当前目录
`..` 代表上一级目录

```
root@iZ2zebl4x12ziohbo8yqefZ:~/thu-secondhand# cd ..
root@iZ2zebl4x12ziohbo8yqefZ:~#
```

看到提示符从 `~/thu-secondhand#` 变回了 `~#`，说明回到上一级了。

#### 一次往上多级

```
cd ../..
```

意思：向上两级。

#### 不管在哪，直接跳到指定路径

```
cd /opt/thu-secondhand
```

开头的 `/` 表示"从根目录开始"。这样不管你现在在哪，都会跳到 `/opt/thu-secondhand`。

#### 回到用户根目录

```
cd
```

或者

```
cd ~
```

`~` 代表"当前用户的根目录"，对 root 用户来说就是 `/root`。

---

### 2.4 查看文件内容（cat）

```
cat 文件名
```

**cat** = Concatenate（连接，但实际用途就是看文件内容）

```
root@iZ2zebl4x12ziohbo8yqefZ:~/thu-secondhand# cat chat.html
```

屏幕会直接显示文件的所有内容。如果文件很长，会一口气滚到底。

---

### 2.5 只看文件的开头/结尾（head / tail）

```
# 只看文件前 10 行
head 文件名

# 只看文件后 10 行
tail 文件名

# 指定行数（比如只看前 3 行）
head -n 3 文件名
```

```
root@iZ2zebl4x12ziohbo8yqefZ:~/thu-secondhand# head -n 5 chat.html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
<meta charset="UTF-8">
```

---

### 2.6 边看边翻页（less）

```
less 文件名
```

如果文件很长，`cat` 会一闪而过。用 `less` 可以慢慢翻：

- 按 `↓`（方向键下）→ 下一行
- 按 `↑`（方向键上）→ 上一行
- 按 `空格` → 下一页
- 按 `q`（键盘 Q 键）→ 退出

---

### 2.7 完整操作示例

假设你刚 SSH 连上服务器：

```bash
# 1. 先看看自己在哪
pwd
# 输出：/root

# 2. 看看有什么
ls
# 输出：snap  thu-secondhand

# 3. 进入项目
cd thu-secondhand

# 4. 看看项目里有什么
ls -lh
# 输出：backend  chat.html  docs  prompt

# 5. 进入后端
cd backend

# 6. 看看后端文件
ls -lh
# 输出：api  data  main.py  ...

# 7. 看看数据库多大
ls -lh data/items.db
# 输出：-rw-r--r-- 1 root root 28K Jul  5 10:30 data/items.db

# 8. 看看系统提示词
cat ../prompt/system_prompt.md

# 9. 看看运行日志
journalctl -u thu-secondhand -n 10 --no-pager

# 10. 退出服务器
exit
```

## 三、SCP命令——下载文件到本机

### 3.1 什么是SCP？

**SCP** = Secure Copy（安全复制）

和 SSH 是"兄弟"关系：
- **SSH** = 远程操作电脑（看文件、执行命令）
- **SCP** = 远程传输文件（下载、上传）

### 3.2 在哪输入SCP命令

**在你本机的 PowerShell 里输入**，不要 SSH 连上服务器！

```
# 本机 PowerShell（❗ 正确）
C:\Users\dengyi> scp ......

# SSH 远程终端（❌ 错误）
root@iZ2zebl4x12ziohbo8yqefZ:~# scp ......  ← 这样不行！
```

### 3.3 下载单个文件到电脑

```
scp root@39.106.1.145:/opt/thu-secondhand/backend/.env C:\Users\dengyi\Desktop\
```

拆开解释：

```
scp          root@39.106.1.145:/opt/thu-secondhand/backend/.env    C:\Users\dengyi\Desktop\
 ↑                      ↑                                               ↑
命令               服务器上的哪个文件                             下载到本机的哪个位置
```

再拆：

```
root@39.106.1.145
  ↑        ↑
用户名   服务器的IP
  分隔符（@）

:/opt/thu-secondhand/backend/.env
  ↑                     ↑
冒号分隔              文件在服务器上的完整路径
```

**所以整体意思是：** 用 SCP 命令，从 IP 为 39.106.1.145 的服务器上，以 root 身份，把 `/opt/thu-secondhand/backend/.env` 这个文件，下载到你电脑的桌面上。

### 3.4 下载整个文件夹

```
scp -r root@39.106.1.145:/opt/thu-secondhand/backend C:\Users\dengyi\Desktop\
```

`-r` = recursive（递归），意思是文件夹里的所有东西都要复制。

### 3.5 上传文件到服务器

把来源和目标反过来就行：

```
scp C:\Users\dengyi\Desktop\test.txt root@39.106.1.145:/opt/thu-secondhand/
```

### 3.6 实际常用示例

```bash
# 下载整个项目备份到桌面
scp -r root@39.106.1.145:/opt/thu-secondhand C:\Users\dengyi\Desktop\thu-server-backup

# 下载数据库文件到桌面
scp root@39.106.1.145:/opt/thu-secondhand/backend/data/items.db C:\Users\dengyi\Desktop\

# 上传本地修改过的文件到服务器
scp C:\Users\dengyi\Desktop\thu-secondhand-agent\backend\api\agent.py root@39.106.1.145:/opt/thu-secondhand/backend/api/agent.py
```

### 3.7 SCP 和 SSH 配合使用

实际操作中，你经常需要**来回切换**：

```
窗口1：SSH 连上服务器看文件
  → 找到文件位置
  → 记下路径
  → 退出 SSH

窗口2：在本机 PowerShell 用 SCP 下载
  → scp root@39.106.1.145:记下的路径 本机位置
```

## 四、常用目录路径（快速导航）

### 我们的项目在服务器上的位置

```
/opt/thu-secondhand/                   ← 项目根目录
├── backend/                           ← 后端代码
│   ├── api/                           ← 各功能模块
│   │   ├── agent.py                   ← AI核心（最重要的文件）
│   │   ├── list_item.py               ← 发布商品
│   │   ├── search_item.py             ← 搜索商品
│   │   └── database.py                ← 数据库操作
│   ├── main.py                        ← 程序入口
│   ├── data/
│   │   └── items.db                   ← 数据库文件
│   └── .env                           ← 配置文件（含密码）
├── prompt/
│   └── system_prompt.md               ← AI的岗位说明书
├── chat.html                          ← 测试聊天页面

```

### 查看服务状态和日志

```bash
# 查看服务是否在运行
systemctl status thu-secondhand

# 查看最近 20 条日志
journalctl -u thu-secondhand -n 20 --no-pager

# 实时查看日志（按 Ctrl+C 停止）
journalctl -u thu-secondhand -f

# 重启服务
systemctl restart thu-secondhand
```

## 五、常见问题

### Q：输入密码时屏幕没反应？

正常。Linux 输入密码时为了安全，**什么都不显示**——没有星号、没有光标动。只管继续打字，打完后按回车。

### Q：命令打错了怎么办？

按 `Backspace`（退格键）删掉错的，重新打。或者按 `Ctrl + C` 取消当前行，重新开始。

### Q：卡住了退不出来？

按 `Ctrl + C`。这个是"紧急停止"快捷键，大多数情况都能退出来。

### Q：怎么退出 SSH？

```
exit
```

或者按 `Ctrl + D`。

### Q：`ls` 和 `ls -lh` 有什么区别？

- `ls` = 只显示文件名
- `ls -lh` = 显示文件名 + 大小 + 修改时间 + 权限

`-l` 和 `-h` 叫"参数"（也叫选项、flag），用来修改命令的行为。

### Q：命令里的空格重要吗？

**非常重要。** 命令和参数之间必须有空格。

```
❌ ls-lh      ← 错误的，没有空格
✅ ls -lh    ← 正确的
```

### Q：路径里的斜杠方向？

Linux 服务器用 **正斜杠 /**，不是 Windows 的反斜杠 \。

```
✅ /opt/thu-secondhand/backend/
❌ \opt\thu-secondhand\backend\
```
