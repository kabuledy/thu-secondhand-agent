# ☁️ 云服务器是什么——从头讲明白

## 最简版本

你把代码写在你自己的电脑上，但你的电脑**不能一直开着**（你要睡觉、要带去上课），而且**只有你能用**。

所以需要另一台电脑——**云服务器**。

阿里云服务器 = 阿里公司机房里的**一台电脑**，特点：

1. **一直开机** — 24小时不关
2. **一直联网** — 有固定"电话号码"（IP地址 `39.106.1.145`）
3. **你远程操作它** — 你在自己电脑上发出指令，它在阿里机房执行

---

## 类比理解

| 你的笔记本电脑 | 阿里云服务器 |
|--------------|------------|
| 家里的厨房 | 餐厅的后厨 |

| 没有固定电话 | 有固定电话号码（IP地址） |

## 三个核心问题

### ① 代码怎么"跑"在远程电脑上？

**答：用 SSH 连上去，把文件复制过去。**

就像你有两台电脑，一台在自己家，一台在阿里机房。你把代码通过"网络U盘"（SFTP）复制到阿里那台电脑上。

具体操作（运行这个命令）：

```bash
python sync.py
```

这个脚本做的事：

```
把你电脑上的文件          →   阿里云服务器上的文件
backend/api/agent.py     →  /opt/thu-secondhand/backend/api/agent.py
prompt/system_prompt.md  →  /opt/thu-secondhand/prompt/system_prompt.md
backend/main.py          →  /opt/thu-secondhand/backend/main.py
...
```

复制完后，再发一个命令让程序重启：

```bash
systemctl restart thu-secondhand.service
```

意思是"重新启动程序，让新代码生效"。

### ② 网络请求是怎么处理的？

拆成三个角色讲：

#### 角色1：用户（你）
你在清小搭网页上打字："我想卖自行车"

#### 角色2：清小搭平台
清小搭收到你的话，它要转发给"后端"处理。它知道后端的地址是：

```
http://39.106.1.145/v1/chat/completions
```

于是它往这个地址发了一条消息。

#### 角色3：阿里云服务器
服务器收到了这条消息。但服务器上同时跑着好几个程序，消息得交给**正确的程序**处理。

---

### ③ "交给正确的程序"是怎么做到的？

**靠端口。**

#### 端口是什么？

每台电脑有 **65535 个端口**，就像一栋大楼有 65535 个门。

| 门牌号 | 谁来敲门 | 干什么 |
|--------|---------|--------|
| **80** | 全世界的人 | 普通网页访问（HTTP） |
| **443** | 全世界的人 | 加密网页访问（HTTPS） |
| **5000** | 只有 Nginx 知道 | 我们的 Flask 程序 |

#### 具体处理流程

```
你在清小搭打字："我想卖自行车"
         │
         ▼
清小搭往 http://39.106.1.145/v1/chat/completions 发请求
         │
         ▼
全世界都能访问的 443 号门（HTTPS）收到了请求
         │
         ▼
门卫（Nginx）看了看请求内容：
  "开头是 /v1/ 啊，这是聊天请求"
  "这个应该交给 Flask 处理"
         │
         ▼
门卫（Nginx）把请求转给 5000 号门（Flask）
         │
         ▼
Flask 收到请求，问 main.py：
  "有人找 /v1/chat/completions"
         │
         ▼
main.py 说："这是聊天请求，交给 agent.py 处理"
         │
         ▼
agent.py 说：
  1. 先加载 system_prompt.md（AI的岗位说明书）
  2. 把用户的话 + 岗位说明书 一起发给 DeepSeek
  3. DeepSeek 回复："用户要发布商品，调用 list_item"
  4. 调用 list_item.py（写入数据库）
  5. 生成最终回复
         │
         ▼
回复原路返回：
  agent.py → main.py → Flask → Nginx → CloudFlare → 清小搭 → 你
```

## 用快递公司类比所有概念

| 技术概念 | 类比 |
|---------|------|
| **阿里云服务器** | 京东的仓库（24小时运转） |
| **你的电脑** | 你家（随时会关门） |
| **SSH** | 管理员去仓库的通道 |
| **端口 443** | 仓库的收件窗口（全世界都能投递） |
| **端口 80** | 仓库的普通收件窗口 |
| **端口 5000** | 仓库的内部处理区（不对外开放） |
| **Nginx** | 分拣员——看快递单（URL），决定送哪个部门 |
| **Flask** | 客服部——接收请求，分给不同的人处理 |
| **agent.py** | 高级客服经理——用 AI 处理复杂问题 |
| **list_item.py** | 上架员——把商品信息录入系统 |
| **database.py** | 档案管理员——管理所有记录 |
| **items.db** | 档案柜 |

## 整个项目完整流程图

```
你在清小搭聊天框输入："我想卖自行车"
        │
        ▼
清小搭平台（清华自己的平台）
  用你的话拼成一个 HTTP 请求
  请求地址: http://39.106.1.145/v1/chat/completions
  请求体: {"messages": [{"role":"user", "content":"我想卖自行车"}]}
        │
        ▼ HTTPS 加密通道
        │
CloudFlare
  接收 HTTPS 请求
  解密为普通 HTTP
  转发到阿里云服务器 39.106.1.145:80
        │
        ▼ HTTP
        │
Nginx（门卫，在服务器上监听 80 端口）
  看到 URL 是 /v1/chat/completions
  判断："这是 AI 聊天请求"
  转发到 localhost:5000（Flask 程序）
        │
        ▼
Flask main.py（在 5000 端口）
  路由匹配 /v1/chat/completions
  调用 run_agent()
        │
        ▼
agent.py
  1. 读 system_prompt.md（告诉 AI 它是谁）
  2. 把用户消息 + system prompt 发给 DeepSeek
  3. DeepSeek 分析后返回："用户要发布商品，调用 list_item"
  4. 执行 list_item.py → 写入 items.db（SQLite）
  5. 把结果返回给 DeepSeek
  6. DeepSeek 生成最终回复
        │
        ▼
回复原路返回：Flask → Nginx → CloudFlare → 清小搭 → 你
你看到："好的！请告诉我价格是多少？"
```
